## 一、后端面试里算法考什么

实习后端通常不会考超难竞赛题，更常见是：

1. **基础数据结构理解是否扎实**（list/dict/set/queue）
2. **复杂度意识是否在线**（能不能解释为什么是 O(n)）
3. **编码习惯是否工程化**（边界条件、可读性、命名）
4. **是否能结合业务场景**（分页、去重、TopK、缓存）

## 二、必会数据结构（结合 Python）

## 2.1 List（动态数组）

特点：有序、可重复、支持下标。

- 末尾 `append`：均摊 O(1)
- 按索引访问：O(1)
- 中间插入/删除：O(n)
- 查找某元素：O(n)

适用：
- 保留顺序的数据
- 分页后的结果集

## 2.2 Dict（哈希表）

特点：键唯一、查找快。

- 增删改查平均 O(1)
- 最坏 O(n)（哈希冲突严重）

适用：
- ID 到对象映射
- 计数统计（词频、出现次数）

## 2.3 Set（哈希集合）

特点：元素唯一，自动去重。

- `in` 查询平均 O(1)
- 去重非常高效

适用：
- URL 去重
- 过滤已处理任务

## 2.4 Queue / Deque

Python 常用 `collections.deque`：

- `append` / `appendleft` O(1)
- `popleft` O(1)

适用：
- BFS
- 任务队列、滑动窗口

## 三、时间复杂度（必须能口算）

你至少要熟到“看到代码就能说复杂度”：

- O(1)：字典查询、数组按下标访问
- O(log n)：二分查找
- O(n)：单层遍历
- O(n log n)：高效排序（Timsort/归并/快排平均）
- O(n²)：双重嵌套遍历

### 面试常见追问

**Q：为什么字典是 O(1)？**
A：底层哈希表，键先算哈希定位桶位，平均常数时间；但碰撞严重时会退化。

**Q：排序复杂度为什么不是 O(n)？**
A：比较排序理论下界通常是 O(n log n)，除非特殊条件（计数排序等非比较排序）。

## 四、高频算法模板

## 4.1 双指针（Two Pointers）

适合：有序数组、区间收缩、去重。

```python
def two_sum_sorted(nums, target):
    l, r = 0, len(nums) - 1
    while l < r:
        s = nums[l] + nums[r]
        if s == target:
            return [l, r]
        if s < target:
            l += 1
        else:
            r -= 1
    return [-1, -1]
```

复杂度：O(n) 时间，O(1) 额外空间。

## 4.2 哈希计数（Hash Map）

适合：频次统计、两数之和、首个不重复字符。

```python
def first_unique_char(s: str) -> int:
    cnt = {}
    for ch in s:
        cnt[ch] = cnt.get(ch, 0) + 1
    for i, ch in enumerate(s):
        if cnt[ch] == 1:
            return i
    return -1
```

## 4.3 二分查找（Binary Search）

适合：有序数组查找、边界问题（第一个 >= x）。

```python
def lower_bound(nums, target):
    l, r = 0, len(nums)
    while l < r:
        mid = (l + r) // 2
        if nums[mid] < target:
            l = mid + 1
        else:
            r = mid
    return l
```

## 4.4 滑动窗口（Sliding Window）

适合：子串、连续区间最值、固定窗口统计。

```python
def max_sum_k(nums, k):
    if k > len(nums):
        return 0
    window = sum(nums[:k])
    ans = window
    for i in range(k, len(nums)):
        window += nums[i] - nums[i - k]
        ans = max(ans, window)
    return ans
```

虽然你项目核心是 API + 爬虫，但第九部分可以这样和业务挂钩：

1. **分页查询**：本质是区间处理与结果截断（避免一次性加载全量）

## 六、面试高频问答（可直接背）

### Q1：List 和 Dict 的核心区别？

A：
- List 适合顺序存储，按索引访问 O(1)，按值查找 O(n)。
- Dict 适合键值映射，按键查找平均 O(1)。

### Q2：如何快速判断一个元素是否存在？

A：
- 若只判断存在性，优先 `set` 或 `dict`，平均 O(1)。
- 不建议每次遍历 list（O(n)）。

### Q3：为什么要关注复杂度？

A：
- 输入规模变大时，复杂度决定系统是否还能跑得动。
- 后端接口要考虑高并发与大数据量，复杂度就是成本。

### Q4：你最常用的算法思想是什么？

A：
- 哈希、双指针、二分、滑动窗口。
- 这些在实际业务里出现频率最高。

### Q5：排序一定要手写快排吗？

A：
- 业务开发里通常优先内置排序（稳定、可读、可靠）。
- 面试若要求手写，可以写快排/归并并解释复杂度。

## 七、手写题作答框架（防止卡壳）

你可以按这 5 步回答：

1. **复述题意**：确认输入输出和边界
2. **先给朴素解**：哪怕 O(n²)
3. **再优化**：引入哈希/双指针等降复杂度
4. **写代码**：命名清晰、处理空输入
5. **口头验算**：举 1-2 个样例跑一遍

> 面试官很看重“思考过程表达”，不只是最终 AC。
