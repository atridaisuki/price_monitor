## 三、Python 高频问答（可直接背）

### Q1：GIL 是什么？

A：
- GIL 是 CPython 的全局解释器锁，同一时刻只有一个线程执行 Python 字节码。
- 对 CPU 密集任务，多线程收益有限；I/O 密集任务仍可受益。

### Q2：深拷贝和浅拷贝区别？

A：
- 浅拷贝只拷贝第一层引用，嵌套对象共享。
- 深拷贝递归拷贝所有层级，互不影响。

### Q3：装饰器本质是什么？

A：
- 本质是“接收函数并返回函数”的高阶函数。
- FastAPI 路由装饰器就是典型应用。

### Q4：为什么要类型提示？

A：
- 提高可读性、IDE 智能提示、便于静态检查和团队协作。

## 四、Web/FastAPI 高频问答（可直接背）

### Q1：GET 和 POST 区别？

A：
- GET 通常用于读，参数常在 URL；POST 用于创建，参数在 Body。
- 语义上 GET 应幂等，POST 通常非幂等。

### Q2：401 和 403 区别？

A：
- 401：未认证（没登录/凭证无效）
- 403：已认证但无权限

### Q3：FastAPI 的优势？

A：
- 异步支持好、类型提示友好、自动 OpenAPI 文档、开发效率高。

### Q4：Depends 的作用？

A：
- 依赖注入。把 session、鉴权、公共参数等在路由调用前统一注入。

## 五、数据库/ORM 高频问答（可直接背）

### Q1：索引有什么优缺点？

A：
- 优点：提升查询速度。
- 缺点：占用空间，写入（insert/update/delete）会更慢。

### Q2：事务是什么？

A：
- 一组操作要么全部成功，要么全部失败，满足 ACID。

### Q3：为什么用 ORM？

A：
- 提升开发效率、减少样板 SQL、模型和业务代码更一致。
- 复杂场景仍可结合原生 SQL。

### Q4：`scalar_one_or_none()` 为什么常用？

A：
- 单条查询场景最稳妥，找到返回对象，找不到返回 None，便于显式 404 处理。

## 六、异步与爬虫 高频问答（可直接背）

### Q1：什么时候应该用异步？

A：
- I/O 密集型：数据库、HTTP 请求、文件/网络操作。
- CPU 密集型建议多进程或任务队列。

### Q2：爬虫为什么会失效？

A：
- 页面结构变化、反爬策略升级、网络超时、地区/货币差异。

### Q3：如何提高爬虫稳定性？

A：
- 超时设置、重试退避、异常分层、日志监控、选择器回归验证。

### Q4：如何扩展新平台？

A：
- 继承 `BaseScraper`，实现 `scrape` 和 `is_valid_url`，注册到 `SCRAPER_MAP`。

## 七、工程化问题：Git/Docker/测试/优化（高分区）

### Q1：你如何管理代码协作？

A：
- `master/main` 保持稳定，功能走 `feature/*`，通过 PR 合并，冲突手工解决后回归测试。

### Q2：为什么用 Docker？

A：
- 统一运行环境，降低“本地能跑、线上不行”的问题，部署可复现。

### Q3：你如何保证质量？

A：
- 单测覆盖核心逻辑，集成测试覆盖 CRUD 主流程，关键链路做回归验证。

### Q4：你做过哪些性能优化？

A：
- 分页限制返回规模、异步 I/O 减少阻塞、请求超时控制、防止慢依赖拖垮接口。

## 八、行为面试题（HR+技术面都会问）

### Q1：你遇到过的最大挑战？

回答模板：
1. 背景：Steam 页面结构变化导致解析失败
2. 动作：定位选择器、补异常分类、加回归样本
3. 结果：恢复抓取并降低同类故障定位成本

### Q2：你如何排查线上问题？

回答模板：
1. 先看影响范围和错误率
2. 再按链路拆分（网关/应用/DB/外部依赖）
3. 用日志和指标定位根因
4. 快速止损 + 事后复盘

### Q3：你如何学习新技术？

回答模板：
- 先跑通最小可用 Demo
- 再读官方文档与源码关键路径
- 最后在项目里做小范围实战验证
