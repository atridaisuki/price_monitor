## 一、面试里的性能优化考什么

实习面试不要求你做极限调优，但要看你是否有：

1. **性能意识**：知道哪里可能慢（DB、网络、序列化、外部依赖）
2. **定位能力**：先监控和分析，再改代码
3. **优化顺序**：先高收益低风险，再深层改造
4. **结果意识**：能用指标证明优化有效

---

## 二、性能分析方法论（先定位再优化）

推荐你用这套回答框架：

1. **定义指标**：P50/P95 延迟、QPS、错误率、CPU、内存
2. **定位瓶颈**：日志 + APM + 慢 SQL + 外部请求耗时
3. **制定方案**：数据库、缓存、并发、协议层逐项优化
4. **验证结果**：压测前后对比，确认收益
5. **防止回退**：建立监控告警与回归基线

> 面试加分句：
> “我不会盲目优化，会先量化瓶颈，再做最小改动验证收益。”

## 三、数据库优化（后端最常考）

## 3.1 索引优化

适合建立索引的列：
- 高频过滤条件（`WHERE`）
- 关联键（`JOIN`）
- 排序字段（`ORDER BY`）

注意：
- 索引能提升查询，但会增加写入成本
- 不要给低区分度字段乱建索引

## 3.2 查询优化

典型问题和手段：

1. **全表扫描慢** → 加索引 + 减少 `SELECT *`
2. **N+1 查询** → 预加载/批量查询
3. **返回数据过多** → 分页 + 字段裁剪

你项目里已经有分页实践（`offset/limit`），这是性能优化的基础动作。

## 3.3 事务与连接池

- 事务保持尽量短，避免长事务锁表
- 使用连接池减少频繁建连开销
- 配置合理池大小，防止 DB 被打爆

## 四、API 层优化

## 4.1 响应优化

1. **分页**：避免一次返回海量数据
2. **字段裁剪**：返回必要字段
3. **压缩**：启用 gzip/br（按网关和服务能力）

## 4.2 协议优化

- 合理使用 HTTP 状态码
- 缓存友好头（`Cache-Control`, `ETag`）
- 减少重复请求（前后端协同）

## 4.3 限流与保护

- IP 或用户级限流
- 熔断降级（依赖超时时快速失败）
- 防止雪崩传播

## 五、异步与并发优化

你项目是 FastAPI 异步栈，这本身是性能优势点。

## 5.1 何时用异步

- I/O 密集任务：数据库、HTTP 请求、爬虫抓取
- CPU 密集任务不建议只靠异步，应考虑多进程/任务队列

## 5.2 并发控制

- 不是并发越高越好
- 要结合下游能力（DB、目标站点、第三方 API）设置并发上限

## 5.3 外部请求优化

- 设置超时
- 失败重试（带退避）
- 连接复用

---

## 六、缓存与热点治理

## 6.1 哪些数据适合缓存

- 读多写少
- 热点明显
- 允许短暂不一致

## 6.2 常见缓存策略

- Cache Aside（最常见）
- TTL + 主动失效
- 热点数据预热

## 6.3 热点 Key 风险

- 击穿：单热点过期瞬间并发回源
- 雪崩：大量缓存同时过期

治理：
- 互斥更新
- 过期时间随机化
- 多级缓存

## 七、面试高频问答（可直接背）

### Q1：你如何定位慢接口？

A：
- 先看监控（P95、错误率），再拆分耗时（DB、外部 HTTP、业务处理），结合慢 SQL 和日志定位根因。

### Q2：为什么分页能优化性能？

A：
- 限制单次查询和序列化数据量，减少 DB 压力与网络传输。

### Q3：索引是不是越多越好？

A：
- 不是。索引会占空间并拖慢写入，应该围绕查询模式设计。

### Q4：异步一定更快吗？

A：
- 对 I/O 密集场景通常更好；CPU 密集场景不一定，需要多进程或离线任务。

### Q5：你做过的性能优化案例怎么讲？

A：
- 说清“问题指标 -> 原因定位 -> 改动方案 -> 优化结果”。没有量化结果也要给出可观测指标。

## 八、结合你项目的回答模板

> “在 Price Monitor 项目里，我主要从数据库分页、异步 I/O 和抓取超时控制三个方向做性能保障。列表接口使用 `offset/limit` 避免大结果集；爬虫请求使用异步客户端并设置超时，防止外部依赖拖慢整体；后续如果读流量增长，我会用 Redis 对商品详情和热点列表做缓存，并配合失效策略避免脏读和缓存击穿。整体策略是先监控定位，再做最小改动验证收益。”

## 九、面试前速记清单

1. 能说清性能优化方法论（先测量再优化）
2. 能说清 DB 三板斧（索引、查询、分页）
3. 能说清异步适用场景（I/O vs CPU）
4. 能说清缓存收益与风险（击穿/雪崩）
5. 准备一个“定位 -> 优化 -> 验证”的项目叙述
